<h1 class="part-title">Part III</h1>
<h1 class="part-subtitle">Building With AI Tools</h1>

<div class="callout">
<p><strong>What you'll understand after this part:</strong> How to use AI coding tools — not as autocomplete, but as genuine building partners. How Claude Code's agent architecture works under the hood. How Cursor, Copilot, Gemini CLI, and the no-code builders each solve different problems. How to choose the right tool for your team, your budget, and your workflow. How to set up knowledge systems so AI gets smarter across sessions instead of starting from zero every time.</p>
<p><strong>What you'll build:</strong> A complete AI-assisted development environment. You'll configure Claude Code with a production-grade CLAUDE.md, deploy subagents for parallel work, connect MCP servers to real databases and APIs, and build a reusable skill that automates a workflow you do every week. Then you'll run the same task through two different AI tools and compare — with data, not opinions.</p>
<p><strong>The shift this part demands:</strong> Parts I and II taught you how software and AI work. This part teaches you how to <em>build with AI as your co-pilot</em>. The skill is not typing code faster. The skill is describing what you want with enough precision that an AI agent can build it correctly, and reviewing what it produces with enough depth to catch what it got wrong. This is the new literacy. Every chapter from here forward assumes you're building alongside an AI partner.</p>
